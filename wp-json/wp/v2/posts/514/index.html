Qu’est-ce qu’une application d’une seule page\u00a0?<\/h2>\n\n\n\n

Une application \u00e0 page unique, ou SPA, est une solution Web qui rend le code JS directement dans un navigateur et n’inclut aucun rechargement de page pendant le parcours d’un utilisateur. Son objectif initial est l’optimisation de l’exp\u00e9rience mobile ; SPA cr\u00e9e une exp\u00e9rience de type natif directement dans le navigateur mobile de l’utilisateur. Apr\u00e8s avoir ouvert la page pour la toute premi\u00e8re fois, toutes les donn\u00e9es sont charg\u00e9es automatiquement.<\/p>\n\n\n\n

Remplacement des pages par des \u00e9tats<\/p>\n\n\n\n

Dans les AMP, les utilisateurs basculent entre des pages distinctes pour acc\u00e9der au contenu qu’ils recherchent. Mais dans les SPA, les utilisateurs sont dirig\u00e9s vers diff\u00e9rents \u00e9tats d’une m\u00eame page.<\/p>\n\n\n\n

Cliquer sur un lien ou un bouton dans un SPA ne rechargera pas la page. Le navigateur n’affiche qu’un certain \u00e9tat de la page et, lorsque vous y \u00eates invit\u00e9, seul le contenu demand\u00e9 est modifi\u00e9. Le reste reste le m\u00eame.<\/p>\n\n\n\n

Cette fonctionnalit\u00e9 est possible gr\u00e2ce \u00e0 AJAX, qui permet au client de communiquer avec le serveur sans recharger la page. Nous approfondirons le fonctionnement des SPA un peu plus loin dans cet article<\/p>\n\n\n\n

Consultez Gmail, Trello ou m\u00eame votre fil d’actualit\u00e9 Facebook. Ce sont tous des exemples de solutions Web monopage.<\/p>\n\n\n\n

En parlant d’une application \u00e0 une seule page, nous entendons litt\u00e9ralement qu’une application n’a qu’une seule page. Cette page HTML unique fournit des mises \u00e0 jour dynamiques et r\u00e9duit consid\u00e9rablement la charge du serveur. En cons\u00e9quence, le propri\u00e9taire du site Web profite dans 3 domaines diff\u00e9rents :<\/p>\n\n\n\n

Vitesse de chargement et r\u00e9activit\u00e9 – tout le contenu est t\u00e9l\u00e9charg\u00e9 automatiquement au d\u00e9but afin qu’un utilisateur n’ait pas \u00e0 attendre apr\u00e8s avoir cliqu\u00e9 sur chaque bouton du site Web
Simplicit\u00e9 – un mod\u00e8le SPA cr\u00e9e une exp\u00e9rience lin\u00e9aire qui n’est pas interrompue par des obstacles comme une mauvaise connexion ou un logiciel obsol\u00e8te
Exp\u00e9rience utilisateur – les utilisateurs s’attendent \u00e0 ce que votre site Web r\u00e9agisse \u00e0 leurs requ\u00eates en moins de 2 secondes et si votre site Web r\u00e9pond rapidement, vous r\u00e9pondez \u00e0 leurs exigences et contribuez \u00e0 l’exp\u00e9rience utilisateur globale.<\/p>\n\n\n\n

Avantages des applications \u00e0 page unique<\/h2>\n\n\n\n

Les performances sont une caract\u00e9ristique importante des applications \u00e0 page unique. Ils obtiennent une am\u00e9lioration des performances en chargeant des ressources HTML, CSS et JavaScript d\u00e8s que le site Web est charg\u00e9.<\/p>\n\n\n\n

La raison en est que lorsque les utilisateurs acc\u00e8dent \u00e0 une application, ils ont besoin du temps d’attente le plus court possible pour pouvoir faire leur travail et partir. Les performances refl\u00e8tent la demande de l’application.<\/p>\n\n\n\n

Si l’application ne fournit pas les performances n\u00e9cessaires, les utilisateurs peuvent quitter cette application et choisir une autre plate-forme. C’est l’une des principales raisons pour lesquelles les d\u00e9veloppeurs choisissent aujourd’hui des applications \u00e0 page unique.<\/p>\n\n\n\n
