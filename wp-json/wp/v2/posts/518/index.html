Code source HTML : restez l\u00e9ger !<\/h2>\n\n\n\n

Pour afficher une page Web, le code source HTML est la premi\u00e8re chose que les navigateurs Web t\u00e9l\u00e9chargent, analysent et mod\u00e9lisent. Si la quantit\u00e9 de code contenue dans la page est trop importante, ces \u00e9tapes sont ralenties et le rendu est retard\u00e9.<\/p>\n\n\n\n

Avec notre prochaine mise \u00e0 jour, nous commencerons \u00e0 v\u00e9rifier les tailles de votre page HTML. “Sizes” est au pluriel, car nous v\u00e9rifierons \u00e0 la fois la taille compress\u00e9e (par exemple: gzip) et la taille non compress\u00e9e.

Versions jQuery non prises en charge<\/h2>\n\n\n\n

jQuery 1.X n’est plus pris en charge. Pour les sites utilisant l’une de ces versions, nous recommandons \u2013 au moins \u2013 de passer \u00e0 la derni\u00e8re disponible sur la branche 1.X : jQuery 1.12.
M\u00eame s’il n’est pas pris en charge non plus, il corrige de nombreux bugs et probl\u00e8mes de s\u00e9curit\u00e9. N\u00e9anmoins, la solution id\u00e9ale serait de passer \u00e0 la derni\u00e8re version majeure (ou d’utiliser un autre framework \u00e0 jour).<\/p>\n\n\n\n

S\u00e9curisation des liens relatifs avec l’\u00e9l\u00e9ment de base et la directive CSP base-uri<\/h2>\n\n\n\n

Nous avons beaucoup \u00e9crit sur la politique de s\u00e9curit\u00e9 du contenu sur ce blog afin de prot\u00e9ger vos visiteurs des effets d’une attaque XSS.
Des pages contenant des liens relatifs pourraient \u00eatre d\u00e9tourn\u00e9es lors d’une telle attaque. Nous conseille d’utiliser l’\u00e9l\u00e9ment pour les pages utilisant des liens relatifs. Ce n’est qu’une premi\u00e8re \u00e9tape, et un attaquant pourrait toujours modifier la valeur de base s’il y a une violation XSS (ou ajouter une autre balise ).
Pour \u00e9viter cela, vous pouvez appliquer la valeur de l’attribut href en utilisant la directive CSP base-uri.<\/p>\n\n\n\n

La plupart des logiciels de synth\u00e8se vocale ont des probl\u00e8mes lorsqu’ils essaient de vocaliser des \u00e9l\u00e9ments HTML vides. Nous ajouterons de nouveaux contr\u00f4les de qualit\u00e9 pour d\u00e9tecter les balises HTML probl\u00e9matiques.<\/p>\n\n\n\n

Fichiers GIF, \u00e0 utiliser \u00e0 bon escient<\/h2>\n\n\n\n

Vous avez peut-\u00eatre d\u00e9j\u00e0 vu notre recommandation d’utiliser les formats PNG ou JPEG pour les images GIF non anim\u00e9es. Nous allons ajouter un nouveau ch\u00e8que, cette fois sur GIF anim\u00e9.
